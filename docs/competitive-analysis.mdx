## Detailed Commentary

### vs. [Photon](https://www.photonengine.com/) / [GameLift](https://docs.aws.amazon.com/gameliftservers/latest/flexmatchguide/match-intro.html#match-intro-features)
[Photon](https://www.photonengine.com/) and [AWS GameLift](https://docs.aws.amazon.com/gameliftservers/latest/flexmatchguide/match-intro.html#match-intro-features) are excellent for state synchronization, but they provide zero help when it comes to **on-chain wagering**. If you want to escrow $10 of AVAX for a match, you have to build your own bridge and trust that your centralized server isn't hacked or manipulated to send funds to the wrong person. AMP automates this trust with attestations.

### vs. Pure Smart Contract Games
Building a game purely on-chain (e.g., using MUD or Dojo) is revolutionary but currently limited by the transaction throughput of the underlying L1/L2. AMP allows you to play the game at 60 FPS while getting the same finality and security of a smart contract for the outcome.

### The AMP Advantage
AMP is **engine-agnostic** and **chain-native**. We don't try to build your game for you; we build the trust layer that allows your game to have real economic consequences.
