# Roadmap

Our mission is to become the default settlement layer for all competitive games on Avalanche.

## 2026 Milestones

### Q1: The Foundation (Completed)
- [x] Core Rust `amp-server` (Matchmaker & Verifier) with Cap'n Proto RPC.
- [x] Solidity `AMPRegistry` + `AMPSettlement` contracts on Fuji with Ownable2Step, Pausable, ERC2771Context.
- [x] Match lifecycle: create, join, cancel, expire, dispute resolution.
- [x] Challenge-response authentication (`requestChallenge` + `login`).
- [x] `IndexedQueue` matchmaker with MMR-sorted buckets, binary search, 50ms tick.
- [x] Native SDKs: C++, C#, Go, Python, Rust.
- [x] `amp-relayer` with sled-backed queue, EIP-1559 gas, nonce tracking, retry + dead-letter.
- [x] Docker + docker-compose for all services.
- [x] 32 Forge tests (17 Registry + 15 Settlement).
- [x] 30 Rust tests passing.
- [x] CI with cargo fmt, clippy, test, and Forge test.
- [x] Load testing tool (`amp-loadtest`).

### Q2: Developer Beta & Hardening (In Progress)
- [ ] OpenTelemetry integration for `amp-server` and `amp-relayer`.
- [ ] Prometheus `/metrics` HTTP endpoint.
- [ ] Invariant/fuzz tests for contracts (`AMPInvariant.t.sol`).
- [ ] `forge coverage` gate (>90%).
- [ ] 72-hour soak test at 1K concurrent connections.
- [ ] napi-rs Node.js/TypeScript SDK.
- [x] Per-language example projects (`amp-sdk/examples/`).
- [ ] Developer Dashboard for game registration.

### Q3: Mainnet & Decentralization
- [ ] **AMP Mainnet Launch** on Avalanche C-Chain.
- [ ] Introduction of the **Verifier Whitelist DAO**.
- [ ] Support for ERC-20 staking (USDC, JOE, etc.).
- [ ] Open-source Verifier node release for third-party hosting.
- [ ] EIP-712 domain separator for typed data signing.

### Q4: The AMP Subnet
- [ ] Transition to a dedicated **Avalanche Subnet** for high-frequency settlement.
- [ ] 10,000+ matches per day target.
- [ ] Fraud Proof / Challenge mechanism for `ASYNC_VERIFIER`.

---

## Known Limitations (Current)
- Ghost matches from disconnected clients (no disconnect detection yet).
- No player deduplication in match queue.
- Glicko-2 NaN validation not yet implemented.
- No persistence schema versioning for sled.

## Long-term Vision (2027+)
Beyond 1v1 and small arena games, AMP aims to support massive-scale competitive play, including Battle Royales and persistent world territorial control, all settled trustlessly on Avalanche.
