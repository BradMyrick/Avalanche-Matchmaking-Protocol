# SDK Overview

AMP provides lightweight, high-performance SDKs for multiple languages. These SDKs handle the complexities of Cap'n Proto RPC, capability management, and cryptographic signing so you can focus on building your game.

## Supported Platforms

| Language | Typical Use Case | Status | Directory |
| :--- | :--- | :--- | :--- |
| **C++** | Unreal / Custom Engines | Beta | `amp-sdk/cpp` |
| **C#** | Unity / Godot | Beta | `amp-sdk/csharp` |
| **Go** | Server-side / CLI | Beta | `amp-sdk/go` |
| **Python** | Data / AI / Research | Beta | `amp-sdk/python` |
| **Rust** | Performance Tools | Beta | `amp-sdk/rust` |

## Authentication Flow

All SDKs use a challenge-response authentication flow:

1. **Request Challenge**: Client calls `requestChallenge(gameId)` on the `GameSessionService`.
2. **Sign Challenge**: Client receives a nonce and signs it with their wallet private key.
3. **Login**: Client calls `login(gameId, signedChallenge)` with the signed nonce.
4. **Session**: Server returns a `UserSession` capability for matchmaking and match operations.

```rust
// Example (Rust SDK)
let challenge = client.request_challenge(game_id).await?;
let signature = wallet.sign(&challenge.nonce);
let session = client.login(game_id, &signature).await?;
```

<Warning>
The `requestChallenge` RPC (method `@1` on `GameSessionService`) must be called before `login()`. Challenges expire after 60 seconds.
</Warning>

## Installation

The AMP SDK folder is organized by language, with generated bindings and examples for each.

### Unity / .NET (C#)
1. Import the files from `amp-sdk/csharp/generated` into your project.
2. Consult the example in `amp-sdk/examples/csharp`.

### Native C++
1. Include the headers from `amp-sdk/cpp/include/amp` and `amp-sdk/cpp/generated`.
2. Use the `CMakeLists.txt` provided in `amp-sdk/examples/cpp` as a reference.

### Go
```bash
go get github.com/avalanche-matchmaking-protocol/amp-sdk/go
```

### Python
```bash
pip install pycapnp
```
Use the schemas in `amp-sdk/python/generated` with `pycapnp`.

### Rust
Add to `Cargo.toml`:
```toml
[dependencies]
amp-sdk = { path = "../amp-sdk/rust" }
```

## Core SDK API

### `AmpClient` / `AMPClient`
The top-level client that connects to an AMP server via Cap'n Proto RPC. Provides `request_challenge`, `login`, `close`, and factory methods for sessions.

### `UserSession`
Created upon successful login. Used for global actions like matchmaking and reconnection.

### `MatchSession`
A short-lived capability granted when a match is found. Used to submit outcomes, emit game events, and stream telemetry.

### `RuleSetBuilder`
A fluent builder for constructing matchmaking rule sets with skill windows, latency limits, and backfill configuration.

### `ProfileBuilder`
A fluent builder for constructing player profiles with MMR data and display metadata.

### `UpdateMMR` (Go / Rust)
A local Glicko-2 rating implementation for client-side skill tracking.

## Versioning Philosophy

The AMP SDKs follow strict semantic versioning. Because the protocol relies on **Cap'n Proto**, the RPC interfaces are backwards and forwards compatible. You can usually update your game server without forcing an immediate update to all deployed game clients.
