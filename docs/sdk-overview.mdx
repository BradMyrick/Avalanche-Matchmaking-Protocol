# SDK Overview

AMP provides lightweight, high-performance SDKs for multiple languages. These SDKs handle the complexities of Cap'n Proto RPC, capability management, and cryptographic signing so you can focus on building your game.

## Supported Platforms

| Language | Typical Use Case | Status | Directory |
| :--- | :--- | :--- | :--- |
| **C++** | Unreal / Custom Engines | Beta | `amp-sdk/cpp` |
| **C#** | Unity / Godot | Beta | `amp-sdk/csharp` |
| **Go** | Server-side / CLI | Beta | `amp-sdk/go` |
| **Python** | Data / AI / Research | Alpha | `amp-sdk/python` |
| **Rust** | Performance Tools | Alpha | `amp-sdk/rust` |

---

## Installation

The AMP SDK folder is organized by language, with generated bindings and examples for each.

### Unity / .NET (C#)
1.  Import the files from `amp-sdk/csharp/generated` into your project.
2.  Consult the example in `amp-sdk/examples/csharp`.

### Native C++
1.  Include the headers from `amp-sdk/cpp/generated`.
2.  Use the `CMakeLists.txt` provided in `amp-sdk/examples/cpp` as a reference.

### Other Languages
- **Go**: Use the generated code in `amp-sdk/go/generated`.
- **Python**: Use the schemas in `amp-sdk/python/generated` with `pycapnp`.
- **Rust**: Use the library in `amp-sdk/rust`.

## Core SDK API

### `UserSession`
Created upon successful login. Used for global actions like matchmaking and MMR lookups.

### `MatchSession`
A short-lived capability granted when a match is found. This is where you stream telemetry and submit outcomes.

### `TelemetryBuffer`
A helper class provided in the SDK to automatically batch and hash your match transcript on a separate thread to ensure zero impact on game frame rates.

---

## Versioning Philosophy

The AMP SDKs follow strict semantic versioning. However, because the protocol relies on **Cap'n Proto**, the RPC interfaces are backwards and forwards compatible. You can usually update your game server without forcing an immediate update to all deployed game clients.
